CSS

  • As portrayed in the preceding image, a selector defines the element or attributes to which the rules, or declarations, will apply.
  • Declarations contain two important components: the CSS property we want to apply and the value of the property. For example, the declaration color: blue; includes the CSS property color and the value for that property, blue.
  • Margin is Outside and Element, Padding is Inside and Element
  • "*" selector applies rule to all elements
  • "." = class selector
  • the "border" property assigns a border

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it

Pushing changes to Git

  • Make sure you are on the correct branch.
  • git status
  • We use the "add -A" command to commit all files in the working branch.
  • git add -A
  • We use the -m flag to associate a message with our commit. Commit messages should be short descrilitive messages of the changes implemented. Once completed, our Git repository is now up to date with our working directory!
  • git commit -m "brief description of changes via commit text"
  • Make sure our current Branch is Up to Date
  • git pull origin main
  • It is a good habit to always pull the base branch into your feature branch before pushing up any code.
  • Push all Changes
  • git push origin "branch"

JavaScript

  • The first variable keyword we might use is "const". The const keyword allows us to create variables that can't be reassigned a value. It essentially "locks down" what that variable will be, and once we declare it, we can never reassign it.
  • We can use the console.log method to output a message to the web console by adding an argument.
  • put "/* & */ before and after the block of code to comment out multiple lines without deleting.
  • A for loop contains 3 important statements, the starting point, condition, and the final statement.
  • Example: for(starting point; condition; finalstatemnt) then {to run the code}
