Git

  • git status
    • Checks what branch we are currently on
  • git checkout -b "branch-name"
    • Creates a new branch and switches to it

Pushing Changes To Git

  • git status
    • Make sure you are on the correct branch.
  • git add -A
    • We use the "add -A" command to commit all files in the working branch.
  • git commit -m "brief description of changes via commit text"
    • We use the -m flag to associate a message with our commit. Commit messages should be short descrilitive messages of the changes implemented. Once completed, our Git repository is now up to date with our working directory!
  • git pull origin main
    • It is a good habit to always pull the base branch into your feature branch before pushing up any code.
  • git push origin "branch"
    • Push all Changes

  • We can use the console.log method to output a message to the web console by adding an argument.
  • put "/* & */ before and after the block of code to comment out multiple lines without deleting.

  • To delcare a function we use the function keyword and the give our funtion a unique name followed by paraentheses. Then we add curly braces that contain the code for the teask we want to perform.
